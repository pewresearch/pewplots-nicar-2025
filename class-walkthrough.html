Theme function

The three parameters that were going to specify are legend, axes, and gridlines. If your style guide makes any of these irrelevant to you, you can delete them and/or specify the default behavior yourself. This function will not control plot size or colors.

Were going to go through the chunks one by one, but they wont run separately. Follow along in nicar_theme.R to build, customize and run your own function.

The first part of our function is setting up the base size and font to be used throughout, the overall header styling, and header styling for facetted plots. Fill in the blanks with your own font. You should feel free to tweak the sizing and margins (which control how far from the plot area elements sit), but you dont need to.

my_theme <- function(legend = FALSE, axes = FALSE, gridlines = FALSE, ...){
  
  #base font size, in pts

  #headline
  plot_title <- element_text(family = "___",
                             size = 13,
                             margin =  margin(b=4, unit = "pt"),
                             #left aligned; for center, hjust = 0.5
                             hjust = 0, vjust = 0)
  #subheadline
  plot_subtitle <- element_text(family = "___",
                                size = 9,
                                #italic, bold?
                                face = "___",
                                margin =  margin(b = 6, unit = "pt"),
                                #left aligned; for center, hjust = 0.5
                                hjust = 0, vjust = 0)
  
  #facet formatting
  strip_background <- element_blank()
  strip_text <- element_text(family = "___", size = 8.5)

Now, we can move on to specifying the parameters were allowing users to interact with. Again, you can set different defaults or allow users to control more/fewer things.

Lets start with legend. Think about what needs to be specified in the TRUE condition, and then specify those objects as NULL/none/element_blank() in the FALSE condition. Youll see why when we put everything together at the end.

If you want to use any attributes that we dont e.g.legend_title in the TRUE condition you can replace element_blank() with element_text() (which pulls the base font size and family from the start of the function).

if (legend == TRUE) {
    legend_title <-  element_blank()
    legend_margin <-  margin(0,0,0,0)
    legend_position <- "top" #or "bottom", "left", "right"
    legend_box_margin <-  margin(-5,-10, 0,-10) 
    legend_text <- element_text()
  } else {
    legend_position <- "none"
    legend_title <-  element_blank()
    legend_text <-  element_blank()
    legend_margin <- NULL
    legend_box_margin <- NULL
  }

Lastly, well specify our gridlines. We specify these separately from axes because there are situations in which you may want an x-axis but not x-gridlines, for example, but if you wanted to combine them, you could.

The options here are one set, two sets, or none. We dont use the minor gridlines, but you could by replacing element_blank() with element_line(). You can also control the line type and width, and the color, though we imagine most if not all of us will stick with black/grey.

if (gridlines == 1) { #no x gridlines, yes y gridlines

Now that weve created all of those objects, we can put them all together!

All were doing is taking theme_bw() and replacing some parameters with the objects we created. We needed to specify the FALSE conditions the way that we did so that when this part of the function pulls an object, it doesnt run into errors from unspecified arguments.

We recommend naming your objects similarly to the parameter youre specifying, but not exactly the same way. We use the name of the parameter with an underscore instead of a period, for example.

  theme <- theme_bw(base_size = base_size, base_family = base_family) %+replace%
    theme(plot.title = plot_title,
          plot.subtitle = plot_subtitle,
          plot.caption = plot_caption,
          legend.position = legend_position,
          legend.title = legend_title,
          legend.text = legend_text,
          legend.margin = legend_margin,
          legend.box.margin = legend_box_margin,
          axis.line = axis_line,
          axis.line.y = axis_line_y,
          axis.ticks.x =  axis_ticks_x,
          axis.ticks.y =  axis_ticks_y,
          axis.ticks.length =  axis_ticks_length,
          axis.text.x = axis_text_x,
          axis.text.y = axis_text_y,
          axis.title =  axis_title,
          panel.grid =  panel_grid,
          panel.grid.major.y =  panel_grid_major_y,
          panel.grid.major.x =   panel_grid_major_x,
          panel.grid.minor.y =   panel_grid_minor_y,
          panel.grid.minor.x =  panel_grid_minor_x,
          panel.border =  panel_border,
          strip.background =  strip_background,
          strip.text = strip_text,
          complete = TRUE
    )
}

Line function

Now that we have a function that can apply our styles to any ggplot we create, lets create an actual plotting function. Follow along in nicar_line_simple.R.

First, think through what users need to be able to control. You could go even simpler than this for example, if you always want facet_scales to be fixed, you can specify that in the function and not provide it as a parameter users can interact with. You could add more for example, see nicar_line.R in the Github repo for a version that allows users to control point locations and data labels separately.

my_line <- function(data,
                    x,
                    y,
                    color_by = NULL, #user provided
                    color = "___", #your default color, name or hexcode
                    facet_by = NULL, #user provided
                    facet_ncol = 1,
                    facet_scales = "___", #fixed or free
                    points = "___", #your default point style  extremes, all, none?
                    points_at = NULL )

Start by assigning variables so they can be used later in the function. If you include facetting functionality, youll want to tell your function how to group your data up front.

Quosures are essentially a way to store a variable under a placeholder name that wont be evaluated until later in the function. We use quosures wherever users supply their own expression (think: variable name). This is a great resource if you want to learn more: https://adv-r.hadley.nz/quasiquotation.html

  quo_x <- enquo(x)
  quo_y <- enquo(y)
  quo_color <- enquo(color_by)
  quo_facet <- enquo(facet_by)
  
  #if facetting variable provided, group data by that variable.
  if (quo_is_null(quo_facet)) {
    data <- data
  } else {
    data <- data %>%
      group_by(!!quo_facet)
  }

Then, you can add point and label positioning functionality. Here, our options are all, none, extremes (first and last values), and specific (wherein users can provide a character vector of specific x values to plot points for). You could include more or less than that.

add_label_position() allows us to dynamically set the horizontal and vertical placement of data labels, which prevents them from overlapping with points or going off of the plot area. Then, we use pipes to filter the data based on what users specify they want to plot, and add our label position function.

add_label_position <- function(df) {
    mutate(df,
           label_hjust = case_when(!!quo_x == max(!!quo_x) ~ -0.5, 
                                   !!quo_x == min(!!quo_x) ~ 1.5, 
                                   TRUE ~ 0), 
           label_vjust = case_when(!!quo_x == max(!!quo_x) ~ 0,
                                   !!quo_x == min(!!quo_x) ~ 0,
                                   TRUE ~ -1))
  }
  
  #data and positioning for possible label positions
  if (points == "all" | points == "none" | points == FALSE) {
    points_data <- data %>%
      add_label_position()
  } else if (points == "extremes") {
    points_data <- data %>%
      filter(!!quo_x == min(!!quo_x) | !!quo_x == max(!!quo_x)) %>%
      add_label_position()
  } else if (points == "specific") {
    points_data <- data %>%
      add_label_position() %>%
      filter(as.character(!!quo_x) %in% points_at)
  }

Once weve done our data manipulation steps, we can move on to actual plotting layers using ggplot. First, well create our line layer with geom_line, using the quosures we created above. To evaluate a quosure, we use the expression !!.

If users dont provide a color_by variable, that means well just have one line in our chart and it will default to the color parameter (eitehr your default or what the user provided). If they do, well want to group our data by that variable (which is what the group = !!quo_color line does).

  if (quo_is_null(quo_color)) {
    geom <- geom_line(aes(x = !!quo_x,
                          y = !!quo_y,
                          group = 1),
                      color = color,
                      data = data,
                      linewidth = 1.5)
  } else {
    geom <- geom_line(aes(x = !!quo_x,
                          y = !!quo_y,
                          group = !!quo_color,
                          color = !!quo_color),
                      data = data,
                      linewidth = 1.5)
  }

Well build our point layer similarly, using geom_point. In the data argument, use the points_data object we created in the first step thats how the function will plot just the points users want and not every point in the full data frame.

if (points == FALSE | points == "none") {
    point <- NULL
  } else if (quo_is_null(quo_color)) {
    point <- geom_point(aes(x = !!quo_x,
                            y = !!quo_y),
                        data = points_data,
                        shape = 21,
                        size = 1.5,
                        stroke = 1,
                        fill = "white", 
                        color = color)
  } else {
    point <- geom_point(aes(x = !!quo_x,
                            y = !!quo_y,
                            color = !!quo_color),
                        data = points_data,
                        shape = 21,
                        size = 1.5,
                        stroke = 1,
                        fill = "white")
  }

Now, well build our text layer with geom_text using the quosures, points_data object, and label positioning objects we created earlier in the function. This is how well get data labels onto the nodes.

    labels <- geom_text(aes(x = !!quo_x,
                            y = !!quo_y,
                            label = !!quo_y,
                            hjust = label_hjust,
                            vjust = label_vjust),
                        data = points_data,
                        fontface = "___", #your style
                        family = "___", #your font
                        size = 3)

Facetting is a simpler yes/no situation, where the facet object is NULL if the argument is unused. If users specify a variable to facet by, the else statement will also provide the scales and number of columns to sort facets into.

if (quo_is_null(quo_facet)) {
    facet <- NULL
  } else {
    facet <- facet_wrap(quo_facet,
                        scales = facet_scales,
                        ncol = facet_ncol)
  }

If you want a capped axis, add this line of code.

  axis_style <- coord_capped_cart(bottom = capped_horizontal())

  list(geom, point, labels, facet, axis_style)
